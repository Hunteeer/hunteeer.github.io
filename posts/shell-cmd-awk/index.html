awk 是处理文本和数据的命令行工具，是几乎所有的 Unix-like 系统的标准工具。实际上 awk 拥有自己的语言： AWK 程序设计语言，语法类似 C 语言，它具备了一个完整的语言所应具有的几乎所有精美特性。它的名字来自三位创始人名字的首字母：Alfred Aho、Peter Weinberger、Brian Kernighan。我对 awk 最直观的体会是它可以像处理表格一样处理文本，从文本中提取信息或是将文本格式化输出，只需几行简短的命令即可完成。

作为一门语言，awk 整个体系也是挺庞大的，awk 的 GNU 版本的用户指南 The GNU Awk User’s Guide 非常得详细也很长。我觉得如果是以学习 shell 为目的学习 awk，那么就没必要去了解 awk 的方方面面。从 shell 的角度出发，把它当做普通命令来学即可。本文站在这个角度上。

初试 awk

awk 对被处理的文本作一个直观的划分：记录（Record）和域（Field）。将每一行划分为一条记录，用空白符将记录划分为域，$1, $2, … 按次序分别代表各个域，$0 代表整条记录。

用 netstat 命令获取的部分网络连接信息为例：

$ netstat > ./test-netstat; cat ./test-netstat
Proto Recv-Q Send-Q  Local Address          Foreign Address        (state)
tcp4       0      0  192.168.1.233.65432    45.67.89.100.63485     ESTABLISHED
tcp4       0      1  localhost.60000        localhost.60001        ESTABLISHED
tcp4       0      0  localhost.60001        localhost.60000        ESTABLISHED
tcp4       0      0  192.168.1.233.62342    117.42.134.45.61232    TIME_WAIT
tcp4       0      2  localhost.63771        localhost.63243        ESTABLISHED
tcp4       0      0  192.168.1.233.63441    183.45.74.65.https     CLOSE_WAIT
tcp4       0      0  192.168.1.233.63440    116.24.75.132.https    CLOSE_WAIT
tcp4       0      0  192.168.1.233.62003    123.45.67.189.62387    ESTABLISHED

用 awk 取出其中的第 1、6 列：

$ cat test-netstat | awk '{print $1, $6}'
Proto Foreign
tcp4 ESTABLISHED
tcp4 ESTABLISHED
tcp4 ESTABLISHED
tcp4 TIME_WAIT
tcp4 ESTABLISHED
tcp4 CLOSE_WAIT
tcp4 CLOSE_WAIT

注意到第一行标题明显不对劲，这也说明 awk 确实在碰到空白符时做一个划分。

基本命令格式

awk 基本的格式如下：

pattern {action}

称为 “pattern-action statement”。pattern 部分是匹配条件，action 是对匹配到的记录执行的命令。pattern 如果为空，则匹配所有的记录；action 如果为空，则默认执行 print $0 命令。上一节的例子中由于没有定义 pattern，就对所有行执行了打印第 1 和第 6 个域。

pattern {action} 最外面要使用单引号。

内建变量

把标题行这条记录去掉，同时在每条记录前面加上行号：

$ awk 'NR != 1 {print NR, $1, $6}' test-netstat
2 tcp4 ESTABLISHED
3 tcp4 ESTABLISHED
4 tcp4 ESTABLISHED
5 tcp4 TIME_WAIT
6 tcp4 ESTABLISHED
7 tcp4 CLOSE_WAIT
8 tcp4 CLOSE_WAIT
9 tcp4 ESTABLISHED

awk 的内建变量可以帮助我们更容易地处理文本和数据。NR 表示每条记录的序号即行号，NR != 1 这个 pattern 使得 awk 不处理序号为 1 的记录，第一行也就不会被打印出来了。awk 的内建变量如下：

  • NR：当前记录的序号
  • NF：当前记录中域的总数
  • FNR：在当前文件中当前记录的序号
  • FILENAME：当前被处理的文件名
  • RS：记录分隔符。默认为换行符
  • FS：域分隔符。默认为空白符
  • ORS：输出的记录分隔符。默认为换行符
  • OFS：输出的域分隔符。默认为空格
  • OFMT：输出的数字格式。默认为 %.6g
  • CONVFMT：转换数字时使用的格式。默认为 %.6g

pattern

特殊的 pattern：BEGIN、END

BEGIN {action}
pattern {action}
END {action}

BEGIN {action} 是一开始就执行的动作，之后对每一条记录执行 pattern {action} 进行匹配和操作，等到完成后执行 END {action}。

$ awk 'BEGIN {print "Foreign Address \t(state)"} $6 == "ESTABLISHED" {print $5,"\t", $6} END {printf("File name is: %s", FILENAME)}' test-netstat
Foreign Address 	(state)
45.67.89.100.63485   ESTABLISHED
localhost.60001 	 ESTABLISHED
localhost.60000 	 ESTABLISHED
localhost.63243 	 ESTABLISHED
123.45.67.189.62387  ESTABLISHED
File name is: test-netstat

Statemensts 之间可以用空格、分号或换行符分割。上面的这条命令会先打印标题行，然后筛选出第 6 个域为 “ESTABLISHED” 的记录并打印第 5、6 域。最后打印文件名。

多个匹配条件的逻辑关系

多个匹配条件可以使用逻辑关系与、或、非连接，和 C 语言一样使用 &&, ||, ! 表示。例如打印标题行和行号为偶数的记录：

$ awk 'NR == 1 || NR % 2 == 0 {print NR, $0}' test-netstat
1 Proto Recv-Q Send-Q  Local Address          Foreign Address        (state)
2 tcp4       0      0  192.168.1.233.65432    45.67.89.100.63485     ESTABLISHED
4 tcp4       0      0  localhost.60001        localhost.60000        ESTABLISHED
6 tcp4       0      2  localhost.63771        localhost.63243        ESTABLISHED
8 tcp4       0      0  192.168.1.233.63440    116.24.75.132.https    CLOSE_WAIT

C 语言中大部分运算符在 awk 中是通用的，比如

+ -  * / % ^ ! ++ -- += -= *= /= %= ^= > >= < <= == != ?:

如果直接用逗号连接多个 pattern，则不仅会匹配 pattern1 和 pattern2，在 pattern1 匹配的记录之后和 pattern2 匹配的记录之前的记录也会被匹配。

$ awk 'NR % 3 == 0, NR % 7 == 0 {print NR, $0}' test-netstat
3 tcp4       0      1  localhost.60000        localhost.60001        ESTABLISHED
4 tcp4       0      0  localhost.60001        localhost.60000        ESTABLISHED
5 tcp4       0      0  192.168.1.233.62342    117.42.134.45.61232    TIME_WAIT
6 tcp4       0      2  localhost.63771        localhost.63243        ESTABLISHED
7 tcp4       0      0  192.168.1.233.63441    183.45.74.65.https     CLOSE_WAIT
9 tcp4       0      0  192.168.1.233.62003    123.45.67.189.62387    ESTABLISHED

第 1 个 pattern 匹配第 3、6、9 行，第 2 个 pattern 匹配第 7 行，由于逗号的存在，第 4、5 行也被匹配。

使用正则表达式进行匹配

结合正则表达式实现更强大的匹配。

$ awk 'NR == 1 || $5 ~ /localhost/' test-netstat
Proto Recv-Q Send-Q  Local Address          Foreign Address        (state)
tcp4       0      1  localhost.60000        localhost.60001        ESTABLISHED
tcp4       0      0  localhost.60001        localhost.60000        ESTABLISHED
tcp4       0      2  localhost.63771        localhost.63243        ESTABLISHED

~ /RE/ 和 !~ /RE/ 分别表示正则匹配和反向正则匹配。

action

格式化输出

printf 可以实现内容的格式化输出，语法和 C 语言类似。

$ awk 'NR != 1 && $NF !~ /ESTABLISHED/ {printf("Local: %-24sForeign: %-24sState: %s\n", $4, $5, $6)}' test-netstat
Local: 192.168.1.233.62342     Foreign: 117.42.134.45.61232     State: TIME_WAIT
Local: 192.168.1.233.63441     Foreign: 183.45.74.65.https      State: CLOSE_WAIT
Local: 192.168.1.233.63440     Foreign: 116.24.75.132.https     State: CLOSE_WAIT

输出重定向

结合管线命令可以将输出重定向到其他命令。需要注意的是要使用双引号。

$ awk 'NR > 1 && $NF == "ESTABLISHED" {print $5 | "grep -o -E [0-9]+?$"}' test-netstat
63485
60001
60000
63243
62387

利用 awk 抓取 foreign address，然后通过管线导入 grep 抓取端口号。

也可以重定向到文件。例如将状态为 “ESTABLISHED” 的连接重定向到指定文件：

$ awk 'NR == 1 || $6 == "ESTABLISHED" {print > "established"}' test-netstat
$ ls
close-wait   established  test-netstat
$ cat established
Proto Recv-Q Send-Q  Local Address          Foreign Address        (state)
tcp4       0      0  192.168.1.233.65432    45.67.89.100.63485     ESTABLISHED
tcp4       0      1  localhost.60000        localhost.60001        ESTABLISHED
tcp4       0      0  localhost.60001        localhost.60000        ESTABLISHED
tcp4       0      2  localhost.63771        localhost.63243        ESTABLISHED
tcp4       0      0  192.168.1.233.62003    123.45.67.189.62387    ESTABLISHED

直接使用 $6 这个变量作为文件名可实现将不同状态的连接储存到不同的文件：

$ awk 'NR > 1 {print > $6}' test-netstat
$ ls
CLOSE_WAIT   ESTABLISHED  Foreign      TIME_WAIT    test-netstat
$ cat TIME_WAIT
tcp4       0      0  192.168.1.233.62342    117.42.134.45.61232    TIME_WAIT

高级语法

统计不同状态的网路连接的数量：

$ awk 'NR != 1 {states[$6] += 1}; END {for(state in states) printf("%11s: %s\n", state, states[state])}' test-netstat
 CLOSE_WAIT: 2
  TIME_WAIT: 1
ESTABLISHED: 5

注意数组的下标不一定要用数字。

awk 支持的语法有：

if( expression ) statement [ else statement ]
while( expression ) statement
for( expression ; expression ; expression ) statement
for( var in array ) statement
do statement while( expression )
break
continue
{ [ statement ... ] }
expression              # commonly var = expression
print [ expression-list ] [ > expression ]
printf format [ , expression-list ] [ > expression ]
return [ expression ]
next                    # skip remaining patterns on this input line
nextfile                # skip rest of this file, open next, start at top
delete array[ expression ]# delete an array element
delete array            # delete all elements of array
exit [ expression ]     # exit immediately; status is expression

内建函数

awk 内置了一些基本的函数，例如：

  • 数学计算：exp，log，sqrt，sin，cos，atan2
  • 字符串长度：length()
  • 字符串切割：split(s, a, fs)
  • 大小写转换：tolower(str)，toupper(str)

Man 手册里有详细的介绍以及其他函数：

$ man awk

自定义分隔符

/etc/passwd 中使用的分隔符为 “:”，使用以下命令取出其中的用户名及其默认 shell：

$ cat /etc/passwd | awk 'BEGIN {FS = ":"}; NR >= 11 && NR <= 19 {printf("%-18s%s\n", $1, $NF)}'
nobody            /usr/bin/false
root              /bin/sh
daemon            /usr/bin/false
_uucp             /usr/sbin/uucico
_taskgated        /usr/bin/false
_networkd         /usr/bin/false
_installassistant /usr/bin/false
_lp               /usr/bin/false
_postfix          /usr/bin/false

在 action 中可以用 FS = "" 指定分隔符，也可以用参数的形式指定：

$ awk -F ":" 'pattern {action}'

分隔符还可以是一个正则表达式：

$ awk -F "[^a-zA-Z]+" 'pattern {action}'

输出格式的分隔符同样可以指定：

$ cat /etc/passwd | awk 'BEGIN {FS = ":"}; NR >= 11 && NR <= 19 {print $1, $NF}' OFS=" ==> " ORS="\n------\n"
nobody ==> /usr/bin/false
------
root ==> /bin/sh
------
daemon ==> /usr/bin/false
------
_uucp ==> /usr/sbin/uucico
------
_taskgated ==> /usr/bin/false
------
_networkd ==> /usr/bin/false
------
_installassistant ==> /usr/bin/false
------
_lp ==> /usr/bin/false
------
_postfix ==> /usr/bin/false
------

awk 脚本

awk 有自己的语言，是可以作为独立的脚本使用的。尤其对于 statement 或 action 比较多的情况，单独写在文件中会比较方便。

$ cat awk-script
#! /bin/awk -f
BEGIN {
  action
}
pattern {
  action
}
END {
  action
}

将脚本保存为单独的文件，使用 -f 即可调用：

$ awk -f awk-script file
