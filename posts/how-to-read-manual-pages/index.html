Liunx 命令有上千条，逐个记忆是不现实的，幸好有非常好用的 Manual Pages。第一次使用它的时候非常苦恼，因为内容多、看不懂。但是多次使用之后，会发现 Manual Pages 是 Linux 上不可或缺的、人性化的功能之一。

官方文档最靠谱。

1. 如何浏览 Manual Pages

要查阅某个命令的手册页，使用命令：

$ man <command>

信息输出采用的是类似 less 命令的终端分页器。因此只要掌握 less 命令的用法即可自由浏览man页面。简单列一下基本快捷键和用法。

按键 描述
Space 向下翻页
Up/Down 向上翻行/向下翻行
/string/?string 向下搜寻/向上搜寻
n/N 搜寻完成后向下/向上查找下一个
q 退出

man 本身是一个命令，它也有一些高级的用法，那么可以通过下面的命令深入了解man的用法。

$ man man

2. man页面的构成

man 页面一般有一下几个部分。重点关注前几个部分。

  • NAME/名称用途
  • SYNOPSIS/语法摘要
  • DESCRIPTION/完整描述
  • EXAMPLES/举例
  • OPTIONS/选项
  • FILES/相关文件
  • ENVIRONMENT/环境变量
  • SEE ALSO/相关参考
  • HISTORY/历史
  • BUGS/漏洞错误
  • AUTHOR/作者

3. 下达命令的一般规则

Linux 中的命令下达一般遵循相同的规则，名称后面接上选项，例如

$ ls -a # 列出当前目录下的所有文件，包括隐藏文件。单独一个 ls 仅列出可见文件

options/选项

命令选项有两种形式：

  • - 后接单字符。例如 -h
  • -- 后接完整单词。例如 --help

-h 和 --help 意义相同。

一次下达命令可以使用多个选项，用空格隔开。短选项可以合并到一起，例如 ls -al <==> ls -a -l。可想而知，长选项要是合在一起可能会有歧义，因此不能合并。

arguments/参数

一个命令可能有不同的 option，一个 option 可以有自己的 argument。不同选项形式使用 argument 的方式也不同：

  • 短选项：空格隔开即可。例如 mkdir -m 700 dirname
  • 长选项：使用 = 连接选项和参数。 例如 mkdir --mode=700 dirname

4. 读懂 man 页面

4.1 第一行

以 ls 命令为例，man 页面第一行内容如下

$ man ls
LS(1)                     BSD General Commands Manual                    LS(1)

LS后面的(1)表示的是Section 1里的 ls 命令。Manual Pages被划分为8个区段：

Section Description
1 一般使用者可调用命令
2 供系统核心调用的命令
3 常用库函数。大部分为C的函数库
4 设备文件和驱动程序
5 文件格式和说明
6 游戏、屏保
7 杂项、说明文件
8 系统管理员可调用的命令、守护进程

4.2 语法摘要和完整描述

语法摘要主要按照正则表达式的规则，如果了解了正则表达式，就能读懂语法摘要，读懂了语法摘要，基本相当于读懂了man page。简要列举会出现的规则：

规则 描述
[] 方括号内的选项都是可选的
{} 花括号内的选项必须使用
… 参数可以重复添加
| 或的关系，可以选择其中之一
加粗 必须准确无误的输入加粗的部分
斜体/下划线 必须用适当的值代替
绑定 单独的选项和参数组合在一起，则必须同时使用

语法摘要告诉读者如何使用，完整描述则主要告诉读者有何用处。两者配合阅读就能了解命令的用法和作用。

5. 举例

5.1 以 ls 命令为例

$ man ls
LS(1)                     BSD General Commands Manual                    LS(1)

NAME
     ls -- list directory contents

SYNOPSIS
     ls [-ABCFGHLOPRSTUW@abcdefghiklmnopqrstuwx1] [file ...]

DESCRIPTION
	 ...

NAME 简要说明了 ls 命令的作用就是罗列目录里的内容。

ls 后面接了 [-ABCFGHLOPRSTUW@abcdefghiklmnopqrstuwx1]，这些选项都是包含在方括号里，说明是可选的，而且都加了粗，说明必须照原样输入，大写和小写意义是不同的。每个选项色具体作用就要查看后面的 DESCRIPTION。

再后面接的是 [file …]，同样是可选的 argument，加了下划线也就是说需要替换成具体情况下的参数。... 说明可以添加多个 argument。比如，我只要罗列当前目录下名称为 file1、file2 的文件的详细信息：

$ ls -l file1 file2

那么，ls 只会列出这两个文件的详细信息。

5.2 以 find 为例

$ man find
FIND(1)                   BSD General Commands Manual                  FIND(1)

NAME
     find -- walk a file hierarchy

SYNOPSIS
     find [-H | -L | -P] [-EXdsx] [-f path] path ... [expression]
     find [-H | -L | -P] [-EXdsx] -f path [path ...] [expression]

DESCRIPTION
	 ...

[-H | -L | -P] 说明这3个选项是可选的，而且是差不多的。

[-f path]，选项和参数组合在一起，说明两者要同时使用。path 要替换为具体路径。

第一行语法中，path … 未加方括号，则不可缺少这一参数。

[expression]代表用来查找的表达式。
