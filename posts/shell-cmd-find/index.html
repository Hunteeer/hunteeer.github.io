find - 在 Linux 中自如地查找文件

给出一个目录，find 命令能递归地罗列出目录下所有文档和目录，包括子目录下的文档和目录。如果再给出限制条件，就可以筛选出符合要求的文档或目录。如果给出一个文档呢？那么 find 就会在当前工作目录下查找该文档，没有找到会报错。find 支持正则表达式。find 返回的查找结果能传递给下一个命令，作为其参数，由此，find命令可以用来批量处理文档。

Walk a file hierarchy

最基本的命令

$ find <path>

这是 find 最简单的命令，递归地罗列出 <path> 下的所有文档和目录。比较一下 find . 和 find * 会有更好的理解。

这条命令等同于

$ find <path> -print

-print 参数表示将查询结果打印到标准输出，是默认参数。-exec, -ls, -ok 等参数会将该参数覆盖掉，如果没有这些参数，find 就会将查询结果打印出来。

如果在特定目录下找某个文档，可以用 -name 参数，这是最常用到的。比如说要在 /etc 下找 hosts 这个文件：

$ find /etc -name hosts
/etc/hosts

使用 -iname 会忽略大小写。

find 只列出这一个文档，因为只有这个文档符合筛选条件。这些筛选条件称为 “Primary”，暂且翻译为 “预选项。”find 支持非常多的预选项，筛选的方法自然有很多种组合方式。

再举个例子，找到空目录或空文档：

$ find /etc -empty

选项

如果在 Linux 下上面的命令是行得通的，在 macOS 下，你可能会发现它根本不起作用。这是因为 macOS 下，/etc 只是一个软链接，用 ls -l 命令查看一下就知道原因了：

$ ls -l /etc
lrwxr-xr-x@ 1 root  wheel  11 Mar 12 12:50 /etc -> private/etc

真实的目录是 private/etc。如果用 find /private/etc -name hosts，那就有用了。find 还有一个默认的 -P 选项，表示不追踪链接，所以上面的命令在 macOS 下就找不到东西。下面的命令就可以：

$ find -L /etc -name hosts
/etc/hosts

man find 给出的 find 语法是

find [-H | -L | -P] [-EXdsx] [-f path] path ... [expression]
find [-H | -L | -P] [-EXdsx] -f path [path ...] [expression]

[-H | -L | -P] [-EXdsx] 这些就是 find 的可选选项。

-P：不跟踪连接，默认选项
-L：跟踪连接
-H：不跟踪连接，除非链接是作为参数的
-E：将 -regex 解释为扩展正则表达式，而不是基本正则表达式
-X：允许 find 安全地配合使用 xargs，跳过名字里有界定符的文件
-d：先列出目录内文件再列出目录本身。find 默认先列出目录本身
-s：按照字典顺序列出文件

操作符

可以对预选项做逻辑操作，拓展筛选的能力。

(expr)：括号内的 expression 具有优先级

!expr：逻辑非，等价于 -not expr

expr1 -and expr2：逻辑与，等价于 expr1 -a expr2，也等价于 expr1 expr2

expr1 -or expr2：逻辑或，等价于 expr1 -o expr2

预选项

前面的内容是预热，下面的内容才是 find 最有意思的地方。预选项就相当于 “Filter”，用来筛选符合条件的文件。find 更高级的用法是将查找结果传递给下一个命令，作为其参数，这样就可以实现进一步操作，或者是批量操作。

按照文件类型查找

下面的命令表示在 /etc下查找名字是 hosts 的链接：

$ find /etc -name hosts -type l

自然是找不到结果的，因为 /etc 下只有名为 hosts 这个文件，并没有名为 hosts 的链接文件。

可选的类型有：

b：块设备。比如硬盘、光驱
c：字符设备。比如鼠标、虚拟终端
d：目录
f：普通文件
l：符号链接
p：FIFO
s：socket

按照文件大小查找

下面的命令表示在 /etc 下查找大小为 2 blocks 的文件

$ find /etc -size 2

默认情况下，-size 后面数字使用的单位是 block，一般磁盘一个 block 的大小是 512 bytes，使用 ls -i 可以查看这个大小，但是使用这个单位往往是不准的。在文末的 Reference 里的那篇文章有更详细的解释。其实用我们常用的单位也是可以的。比如查找 2 KB 的文件：

$ find /etc -size 2k

其他单位还有：

c: bytes
M: megabytes
G: gigabytes
T: terabytes
P: petabytes

当然要找到大小恰好是 2 KB 的文件，当真不容易，实际的应用场景也不多。所以使用 “+”、“-”号可以筛选特定范围。比如查找小于 2 KB 的文件：

$ find /etc -size -2k

结合逻辑操作符可以设置上下限：

$ find /etc -size +2k -a -size -4k

你说 -a 加不加无所谓，那我也是没话说的。

按照时间筛选

Linux 上每个文件都有 3 个时间，分别是

  • Modify time（mtime）：文件内容最后一次被修改的时间
  • Change time（ctime）：文件属性最后一次被更改的时间
  • Access time（atime）：文件最后一次被访问的时间

使用stat 命令可以查看这三个时间：

$ stat -x /etc/hosts
  File: "/etc/hosts"
  Size: 143393       FileType: Regular File
  Mode: (0644/-rw-r--r--)         Uid: (    0/    root)  Gid: (    0/   wheel)
Device: 1,4   Inode: 6485970    Links: 1
Access: Tue May 30 21:27:06 2017
Modify: Sun May 28 15:16:30 2017
Change: Sun May 28 15:16:30 2017

比如我要查找 2 天前（当前时间 20170530）内容被修改的文件，看看能不能把 hosts 找出来：

$ find /etc -mtime 2
/etc/hosts

确实找出来了。默认的单位是天（进位取整 24 小时）。可以使用的单位有：

s: second
m: minute
h: hour
d: day
w: week

可以使用 “+” 和 “-” 表示一个范围。-ctime、-atime 同理。比如说查找有没有 1 小时 30 分钟以内属性被修改的文件：

$ find /etc -ctime -1h30m

说到时间肯定会有一个参考时间点，以上都是拿现在的时间作为参考点。-newer 则可以使用某一文件的时间作为参考点。比如查找比 hosts 的 mtime 更晚的文件：

$ find /etc -newer hosts

-newer 对比的是 mtime，-cnewer 对比 ctime，-anewer 对比 atime。

如果我要把当前文件的 mtime 和参考文件的 ctime 对比怎么办呢？-newer 还可以这么用：

-newerXY file

X 表示要查找文件的时间，Y 表示参考文件的时间，使用以下代号就可以表示不同类型的时间：

  • a: atime
  • c: ctime
  • m: mtime
  • B: inode creation time （真正的文件创建时间）

Y 还有一个特殊的值 t，表示参考文件的时间被解释成 CVS 能理解的时间格式

按照权限查找

查找属于 root 用户的文件：

$ find /etc -user root

同样，-group 可以查找某个用户组的文件。不仅仅可以用名字，ID 也是适用的。

还可以查找权限为 644 的文件：

$ find /etc -perm 644

如果要表示一个范围可以使用 “+”、“-”。“+” 号表示后面的权限只要有一位被满足就行；“-” 号表示后面的权限所有位都至少要被满足。

限制查找的目录深度

只在从当前工作目录数起的第 2 层子目录查找：

$ find /etc -name hosts -depth 2

-depth 0 表示什么？是当前工作目录这一级的目录，只有当前工作目录本身。

如果要限制的是一个范围要怎么办呢？这里就不是使用 “+”、“-” 啦，而是有特定的上下限：-maxdepth、-mindepth。

还有一个预选项：-prune ，表示到当前工作目录就停止，不再递归地找下去了。

$ find /etc -name hosts -prune

对返回值进行处理

看下面这个例子：

$ find /etc -name hosts -ls
6485970      288 -rw-r--r--    1 root      wheel      143393 May 28 15:16 /etc/hosts

这相当于，find 找到结果之后，没有急着打印到标准输出，而是将返回值传递给了 -ls 处理，然后打印出详细信息。

这和使用 xargs 命令的作用类似。比如

$ find /etc -name hosts | xargs ls -gilds

xargs 能将前一个命令的标准输出，通过管线命令，作为后一个命令的参数。find 和配合 xargs 不仅仅可以实现 ls 命令，其他类似的命令也能够实现，这就实现了对返回值的进一步加工。但是呢，这样做在碰到返回的文件路径中包含界定符的时候就不行了，这时候可以思考下最前面选项那里的 -X，可以避免出错，但是仍旧不能处理有界定符的文件。

find 自带了一个参数：-exec，不仅能实现 xargs 的功能，还能避免上面的问题。举个例子：

$ find /etc -name hosts -exec ls -l {} \;
-rw-r--r--@ 1 root  wheel  143393 May 28 15:16 /etc/hosts

{} 会被替换成 find 的返回值，然后作为 -exec 后面接的命令的输入参数，最后不要忘了加上 ;，表示命令结束，因为 ; 在 shell 里有特殊含义，所以要使用转义符 \，或者使用引号。

那能不能用另一个命令的输出作为 find 的输入呢？能，利用反引号。

$ find `pwd` -name file -exec ls -l {} ';'
