test - 条件判断命令

test 命令用于文件类型和属性测试，以及算术、字符串的比较，大部分 Unix-like 系统都支持这个命令。

test 命令是 shell 的内置命令：

$ which test
/bin/test

还有一个叫 [ 的命令和 test 是等效的。两者的用法如下：

test expression
[ expression ]

使用 [ 时，expression 两侧的空格一定不能少，这是因为 [ 是一个命令，命令和参数之间要分隔开。

如果 expression 为真，退出值置为 0；expression 为假，退出值置为 1；发生错误时，退出值大于 1。这和一般 Linux 命令的退出值是一样的。

使用 $? 获取退出值：

$ test 1 -eq 2; echo $?  # 测试 1 等于 2 是否成立
1
$ [ 1 -eq 2 ]; echo $?  # 和上条命令等效
1

文件测试

测试文件是否存在，不论什么类型：

$ test -e /etc/passwd; echo $?
0

测试文件的类型：

-b file  # 是否存在且为块设备文件，即带缓冲的固定大小单元读写的设备文件。比如 /dev/sda
-c file  # 是否存在且为字符设备文件，无缓存的变长单元读写的设备文件
-d file  # 是否存在且为文件夹
-f file  # 是否存在且为普通文件
-L file  # 是否存在且为符号链接
-S file  # 是否存在且为 socket
-p file  # 是否存在且为 named pipe (FIFO)

测试文件的属性：

-r file  # 是否存在且可读
-w file  # 是否存在且可写
-x file  # 是否存在且可执行
-s file  # 是否存在且 size 大于 0
-O file  # 是否存在且 owner 和当前进程的 owner 相同
-G file  # 是否存在且 group 和当前进程的 group 相同

如果在 read-only 文件系统上，即使 test -w file 为真，也不可写。比如 Android 系统内部的文件表面上可写，实际上是修改不了的。

部分 shell 还支持文件之间的比较：

file1 -nt file2  # 如果文件 1 存在且比文件 2 新则为真
file1 -ot file2  # 如果文件 1 存在且比文件 2 旧则为真
file1 -ef file2  # 如果文件 1 和文件 2 都存在且为同一文件则为真

还有些其他的：

-g file  # True if file exists and its set group ID flag is set
-k file  # True if file exists and its sticky bit is set
-u file  # True if file exists and its set user ID flag is set
-t file_descriptor  # True if the file whose file descriptor number is file_descriptor is open and is associated with a terminal

字符串测试

判断字符串是否为 null 或空：

string  # 如果字符串不是 null 则为真
-n string  # 如果字符串长度为非零则为真
-z string  # 如果字符串长度为零则为真

字符串的比较：

s1 = s2  # 字符串相同则为真
s1 != s2  # 字符串不同则为真
s1 > s2
s1 < s2

字符串大小的比较按照字符对应的二进制值的顺序，a-z > A-Z > 0-9。需要注意的是，<, > 在 bash 中还有重定向的作用，因此需要使用 \<, \> 转义。

整数测试

n1 -eq n2  # 是否相等
n1 -ne n2  # 是否不相等
n1 -gt n2  # n1 是否大于 n2
n1 -ge n2  # n1 是否大于等于 n2
n1 -lt n2  # n1 是否小于 n2
n1 -le n2  # n1 是否小雨等于 n2

逻辑运算

! expression  # 非运算
expr1 -a expr2  # 与运算
expr1 -o expr2  # 或运算

例如：

$ [ ! -e /etc/passwd -a 2 -eq 2 ]; echo $?
1

-a 的优先级高于 -o。使用 ( expression ) 提高优先级，但是必须经过转义：

$ [ -d /bin -o -d /etc -a ! -d /var ]; echo $?
0
$ [ \( -d /bin -o -d /etc \) -a ! -d /var ]; echo $?
1

结合 shell 中的 &&, ||，可以更灵活地运用：

$ [ -w /etc/passwd ] && echo '/etc/passwd is writable' || echo '/etc/passwd is not writable'
/etc/passwd is not writable

[ 与 [[

[[ 是 [ 的增强版，相比之下 [[：

  1. 直接使用 <, >, ()，不需要转义；
  2. 使用逻辑运算符 &&, ||，不需要 -a, -o；
  3. 使用等号进行匹配，使用 =~ 甚至支持正则表达式匹配。需要注意的是通配符不能加引号。例如 [[ "abcd" = a*d ]]; echo $?；
  4. [[ 是关键字，并不是一个命令。在 [[ 和 ]] 之间所有的字符都不会发生文件名扩展或者单词分割，但是会发生参数扩展和命令替换。
