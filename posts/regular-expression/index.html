1.2 特殊字符

1.2.1 有特殊含义的字符

字符 描述
^ 字符串开始位置；取反
$ 字符串结束位置
() 分组；定义操作符范围和优先度
[] 从字符集合中选择1个字符
{} 限定符，修饰匹配次数
. 限定符，有且仅有1个字符（除\n ）
* 限定符，有0个或多个
? 限定符，被限定字符出现0次或1次；其他用途
+ 限定符，被限定字符出现1次或多次
转义
| 选择

1.2.2 不可打印的字符

字符 描述 等价
\a 响铃符 \x07
\n 换行符 \x0A
\r 回车符 \x0D
\t 制表符 \x09
\v 垂直制表符 \x0B
\e ESC \x1B
\f 换页符 \x0C

1.3 字符集合

字符 描述
. 任意一个字符（除\n ）
\d 任意一个0-9数字
\w 任意一个字母、数字、下划线
\s 任意一个空白字符

如果正则表达式匹配模式为 SINGLELINE，.可以匹配包含\n在内的任一字符

采用大写字母的作用类似于取反，例如：\D 表示除数字0-9之外的任一字符

2. 限定

2.1 限定符

字符 描述 举例
* 匹配0次或多次 ab* ==> a, ab, abb, …
+ 匹配1次或多次 ab+ ==> ab, abb, …
? 匹配0次或1次 ab? ==> a, ab
{n} 匹配n次 ab{2} ==> abb
{n,} 匹配n次或n次以上 ab{2,} ==> abb, abbb, …
{n,m} 至少匹配n次，至多匹配m次 ab{2,3} ==> abb, abbb

对于次数不确定的匹配，可以有不同选择，默认采用贪婪模式。

模式 用法 描述 举例
贪婪模式 默认 在限定符之后的表达式能够匹配成功的情况下，不定次数的限定符总是尽可能的多匹配 {n,m}
勉强模式 限定符之后添加? 不定次数的限定符总是尽可能的少匹配 {n,m}?
占有模式 限定符之后添加+ 即使之后的表达式匹配失败，不定次数的限定符总是尽可能的多匹配 {n,m}+

2.2 定位符

字符 描述 举例
^ 字符串开始位置 ^a ==> “ab ab”
$ 字符串结束位置 b$ ==>“ab ab“
\b 字符串边界 \bxy\b ==> xyz xy

\B匹配非字符串边界

如果正则表达式的匹配模式为 MULTILINE ，^可匹配一行文本的行首，$可匹配一行文本的行末

3. 分组

使用()将所包含的内容组成一个整体。匹配到的内容可以单独记录，在匹配过程中或结束后被使用。

使用符号表示不同分组的内容，也可以对分组自定义命名。

字符 描述
() 分组
$& 匹配到的内容
$n 组别，n 表示序号1-999
(?pattern) 对某一组进行命名
\n 反向引用，n 表示序号1-999
${name} 指向被命名分组的内容
$$ 表示$本身
$` 原字符串中，匹配到的内容之前的字符串
$’ 原字符串中，匹配到的内容之后的字符串
$+ 所有“有捕获”的分组中，编号最大的那个分组
$_ 代表被替换的整个字符串

举例：

字符 描述
a ab abc Abcd e 原字符串
(\w{2})\s(\w{3})\s(\w(?\w{4})) Pattern
$& ab abc Abcd
$1 ab
$2 abc
$3 Abcd
${No.4} bcd
$` a□
$’ □e

□表示空格

4. 模式

字符 描述
IGNORECASE 忽略大小写
SINGLELINE 单行模式。换行符\n可以被.匹配
MULTILINE 多行模式。^能匹配\n之后的位置,$能匹配\n之前的位置
GLOBAL 全局模式
RIGHTTOLEFT 从右向左匹配
EXTENDED 扩展模式。 忽略表达式中的空白字符，并且把从#开始到该行行末的内容视为注释

5.语法

5.1 注释

字符 描述
(?#xxx) xxx表示的注释内容，特殊符号不起作用，直到遇到)停止

5.2 转义

字符 描述 举例
\ 使特殊符号失去特殊作用 \\ ==> \
\x ASCII 转义 \x22 ==> “
\u Unicode 转义 \u0042 ==> B
\Q…\E 之间的字符全部转为普通字符 \Qa\b?\E ==> a\b?

5.3 选择

字符 描述 举例
x|y 或的关系 a(b|d)c ==> abc, adc
[xyz] 匹配所含字符的任意1个 a[bd]c ==> abc, adc
[a-z] 匹配所含字符范围内的任意1个 a[0-9]b ==> a0b, …, a9b
[^xyz] 匹配所含字符之外的任意1个 a[^0-9]b ==> acb, a.b, …

5.6 预搜索

字符 描述 举例
(?=xxx) 判断当前位置右侧是否能匹配指定表达式 abc(?=})匹配{abc} ==> abc
(?!xxx) 判断当前位置右侧是否不能匹配指定表达式 abc(?!})匹配{abc} ==> 无法匹配
(?<=xxx) 判断当前位置左侧是否能够匹配指定表达式 (?<={)abc匹配{abc} ==> abc
(?<!xxx) 判断当前位置左侧是否不能够匹配指定表达式 (?<!{)abc匹配{abc} ==> 无法匹配

预搜索的内容不作为匹配结果显示，仅作为判断条件

5.4 条件

表达式 条件类型 描述
(?(n)a|b) 数字n 若$n 匹配到内容，则匹配a，否则匹配b
(?(?=x )a|b) 预搜索 若当前位置右侧是x，则匹配a，否则匹配b
(?(name)a|b) 分组命名 若${name} 匹配到内容，则匹配a，否则匹配b

5.5 递归

反向引用是对匹配内容的调用，递归匹配是对包含自身的表达式的调用。
