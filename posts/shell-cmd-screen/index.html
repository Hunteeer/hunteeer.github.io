screen - 命令行窗口管理器

2018-02-28
Linux 的奇妙世界
911
    • 使用 Screen
    • 管理窗口
    • 关闭窗口

之前用 SSH 连接 VPS，执行需要较长运行时间的任务的时候，我就曾陷入这样的僵局：既不能关闭这个窗口，也不能在这个窗口中做其他的事情。Screen 就可以解决这个问题。它相当于一个窗口管理器，能够创建和管理独立于当前窗口的的命令行窗口。把命令放在一个 Screen 窗口中运行，然后将它分离（相当于 daemon 的运行方式），继续回到原来的窗口做其他事情，甚至关闭 SSH 连接。一段时间后再次连接，查看他的运行状态。

使用 Screen

使用命令 screen 就可以新建窗口并进入。使用按键 ctrl + ad 可以从这个窗口分离，回到之前的窗口。在这个窗口运行的命令是独立于之前的窗口的。我来做一个测试。通过 SSH 连接 VPS 后，用 pstree 命令显示进程树：

$ pstree
systemd─┬─agetty
        ├─atd
        ├─chronyd
        ├─crond
        ...
        ├─sshd───sshd───sshd───zsh───pstree
        ...

可以看到 sshd───sshd───sshd───zsh───pstree 显示的进程之间的父子关系，如果断开 SSH 连接，当前 zsh 终端也会被关闭。运行 screen 命令，再次查看：

$ pstree
systemd─┬─agetty
        ├─aria2c
        ├─atd
        ├─chronyd
        ...
        ├─sshd───sshd───sshd───zsh───screen───screen───zsh───pstree
        ...

使用按键 ctrl + ad 分离后，再次查看：

$ pstree
systemd─┬─agetty
        ├─aria2c
        ├─atd
        ├─chronyd
        ...
        ├─screen───zsh
        ├─sshd───sshd───sshd───zsh───pstree
        ...

这时就会发现 screen 已经独立于 SSH，由 Screen 创建的窗口依附于 screen。

可以用 Screen 开启多个相互独立的窗口，将它们罗列出来：

$ screen -ls
There are screens on:
	9377.pts-0.vultr	(Detached)
	9271.pts-0.vultr	(Detached)
2 Sockets in /var/run/screen/S-xhl.

Screen 以 <pid>.<tty>.<host> 的格式命名窗口。再次连接某个窗口：

$ screen -r <pid>

可以给窗口命名以方便再次连接：

$ screen -S sockname

这时 screen -ls 就会以 <pid>.sockname 的格式显示这个窗口，可以用 screen -r sockname 来连接。

直接新建一个分离的窗口：

$ screen -dmS name

管理窗口

进入 Screen 窗口后，要想对窗口本身进行管理，就需要使用 ctrl + a 激活。

  • ctrl + ad：从当前窗口分离
  • ctrl + a?：显示所有可用的按键控制
  • ctrl + ac：创建新的窗口

关闭窗口

  1. 进入 Screen 窗口后，使用 exit 命令退出并清除这个窗口
  2. kill <pid> 直接杀死
  3. screen -S sockname -X quit
