aria2 是目前（20170428）我在各个平台上唯一使用的下载工具。足够，而且非常好用。关于它的使用教程烂大街了，而且感觉都差不太多。。我大概已经使用三个月了，从 Mac，到 VPS 上的 CentOS，到 Android。这次就写写它了。

Introduction

aria2 是一个命令行下载工具，类似于 curl、wget，但是功能更强，亮点是多线程下载、BT 下载。aria 支持的协议/下载源有：

  • HTTP, HTTPS
  • FTP, SFTP
  • BitTorrent
  • Metalink

除了在命令行使用，aria2 支持 RPC 远程控制，利用 VPS 下载外面东西非常方便。

aria2 支持的平台：

  • Windows
  • macOS
  • Android
  • Linux

好吧，直接说全平台一点都不过分。

当然，和一般有 GUI 界面的下载工具比起来，aria2 要麻烦很多，但是目前乘手又良心的下载器真的不多见，最重要的是 aria2 通用性强，一个配置文件，各个平台都可以使用，更加节省时间成本。

相关链接：

基本配置

安装就不必多说了。我写一下 aria2 的三种基本使用方式：

方式 优点
命令行运行
配置文件运行 方便，简化了输入命令的过程
RPC 远程 方便远程下载，下载机

命令行运行

查看 man 页面获取帮助：

$ man aria2c

配置文件运行

下载的时候一般需要配置很多项目，输入命令就比较麻烦了，事先配置好 conf，然后将 aria2 以配置文件的方式启动，方便很多，通用性强。

举个例子，将文档下载到特定目录可以这样：

$ aria2c -d $HOME/download "target url"

那么可以写一个配置文件 aria2.conf：

dir=$HOME/download

然后以配置文件运行 aria2：

$ aria2c --conf-path=<conf path> "target url"

这两种方法效果是一样的。嗯，貌似更加麻烦了。。但是配置项很多的话，那肯定选第二种方式。

有哪些配置项呢？看看 man 里面就知道啦。就是里面的各种选项。

下面是我在 Mac 上使用配置文件，更改目录就可以套用到其他平台了：

#Basic Download
#log=$HOME/.aria2/aria2.log
dir=$HOME/Downloads
continue=true
max-concurrent-downloads=5
max-connection-per-server=8

#Speed Limit
min-split-size=10M
split=10
max-overall-download-limit=0
max-overall-upload-limit=0
max-download-limit=0
max-upload-limit=0

#Save Mission
input-file=$HOME/.aria2/aria2.session
save-session=$HOME/.aria2/aria2.session
save-session-interval=60

#RPC
#rpc-secret=
enable-rpc=true
rpc-allow-origin-all=true
rpc-listen-all=true
rpc-listen-port=6800

各个选项是什么意思看看 man 或者 documentation 就知道了。我上面的配置选项是一般使用需要的基本配置，BT 的没贴。里面的 #RPC 是下面一种方式需要用到的。

RPC 远程

如果对 GUI 界面有需求，或者在服务器、下载机上使用，可以使用这种方式。

  • webui-aria2 - GitHub
  • yaaw - GitHub

这两种比较好用，我现在用的是 webui-aria2，并且在服务器上放了，你可以打开看看：点击链接。

需要在配置文件设置这些选项：

#RPC
#rpc-secret=	#设置 token，外网上面部署肯定需要设置，本地下载就没必要设置了
enable-rpc=true
rpc-allow-origin-all=true
rpc-listen-all=true
rpc-listen-port=6800

然后就可以通过 RPC 远程控制了。可以添加下载，查看进度等。webui 可以设置 aria2 的各个选项，因此直接命令行运行也是可以的。建议还是使用配置文件比较好。

使用情境

这一部分写我在各个平台上使用的具体情境。使用配置文件。

首先，写个运行脚本是有必要的，方便控制。贴一下我在 Mac 上的脚本：

#! /bin/bash

ARIA2C="/usr/local/aria2/bin/aria2c"
CONFPATH="/Users/name/.aria2/aria2.conf"
start() {
	$ARIA2C --conf-path="$CONFPATH" -D
	if [ $? -eq 0 ];then
		echo "aria2 start successfully in RPC mode"
	fi
}

stop() {

	if [ "$(ps -A | grep -m 1 aria2c | awk '{print $4}')" = $ARIA2C ];then
		kill "$(ps -A | grep -m 1 aria2c | awk '{print $1}')"
                echo "aria2 stop successfully"
	else
		echo "aria2 is not running"
    fi
}

status() {
	if [ "$(ps -A | grep -m 1 aria2c | awk '{print $4}')" = "$ARIA2C" ];then
                echo "aria2 is running in RPC mode"
	else
		echo "aria2 is not running"
    fi
}

restart() {
	stop
	sleep 1
	start
}

help() {
	echo -e "choose an option:\nstart\tstop\trestart\tstatus\thelp"
}

case $1 in
start)
	start
	;;
stop)
	stop
	;;
status)
	status
	;;
restart)
	restart
	;;
help)
	help
	;;
*)
	help
	;;
esac

macOS 上的使用

主要就是利用 launchd 配置开机自启，然后就在后台静等 RPC。需要下载的时候打开浏览器，打开 webui 添加下载即可。

在用户家目录下的 Library 里面，具体路径是 $HOME/Library/LaunchAgents。在这个目录里面新建一个 plist，名字就按照 java 的包名规则，内容可以参考其他的 plist。我新建的是 com.name.aria2.plist，内容如下：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
	<key>Label</key>
	<string>com.name.aria2</string>
	<key>ProgramArguments</key>
	<array>
		<string>/Users/name/.aria2/start.sh</string>
	</array>
	<key>RunAtLoad</key>
	<true/>
	<key>StandardErrorPath</key>
	<string>/Users/name/.aria2/error.log</string>
	<key>StandardOutPath</key>
	<string>/Users/name/.aria2/start.log</string>
</dict>
</plist>

然后加载并设置开机自启：

launchctl load $HOME/Library/LaunchAgents/com.name.aria2.plist
launchctl enable com.name.aria2

这样开机就可以自动启动 aria2 了。

可以直接使用 launchctl 来管理，不需要运行脚本。

$ launchctl load aria2
$ launchctl unload aria2
$ launchctl enable aria2
$ launchctl disable aria2
$ launchctl start aria2
$ launchctl stop aria2

CentOS 上的使用

开机启动的配置和 macOS 其实差不多啦，只不过用的是 systemd。

systemd 的单元文件放置在两个地方：/etc/systemd/system/, /usr/lib/systemd/system/。分别放置软件包安装的单元和系统管理员安装的单元。aria2 的放置在 /usr/lib/systemd/system/。目录下新建一个 aria2.service，内容如下，具体的用法可以 man 下（有必要写篇博客，先挖个坑，填坑啦：了解 systemd）：

[Unit]
Description=The next generation download utility
After=network.target

[Service]
ExecStart=/usr/bin/aria2c --conf-path=$HOME/.aria2/aria2.conf

[Install]
WantedBy=multi-user.target

然后设置开机自启：

$ systemctl enable aria2.service

启动试试，并查看状态：

$ systemctl start aria2
$ systemctl status aria2

启动成功。同样可以直接使用 systemctl 来管理 aria2 启动或关闭：

$ systemctl start aria2

$ systemctl status aria2
$ systemctl reload aria2
$ systemctl enable aria2	#设置开机自启
$ systemctl disable aria2	#取消开机自启

这个单元在我本地服务器上测试是有效的，但是在 VPS 上自启失败，手动启动有效。目前的解决方法是添加启动延时，[Service] 里增加这一行 ExecStartPre=/bin/sleep 30

使用 aria2 下载百度云文档

aria2 可以多线程下载，找一个好用的 UA，百度云限速不是问题。但是慎用，可能会被限速。

需要用到 Chrome 的一个插件，整合到百度云界面，可以导出 aria2 下载链接。BaiduExporter - GitHub。作者停更了，但是还能用。
